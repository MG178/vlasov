### 1. Общее описание проекта

Современный рынок курьерских услуг требует высокой скорости обработки заказов, прозрачности перемещения грузов и минимизации человеческого фактора. В существующей практике многих малых логистических компаний заказы принимаются по телефону/WhatsApp, фиксируются в Excel-таблицах, а курьеры отчитываются голосовыми сообщениями. Это приводит к потерям заказов, неверному расчету времени доставки и низкому уровню контроля.

**Цель проекта** — создать современное веб-приложение (клиентская часть на React/Vue/Blazor + серверная часть на C# ASP.NET Core), обеспечивающее:

- автоматизацию приема, обработки и назначения заказов на доставку малогабаритных грузов (до 30 кг);
- маршрутизацию и трекинг перемещения грузов в реальном времени (имитация через смену статусов и фиксацию геолокации);
- подтверждение получения груза через QR-код (цифровая подпись получателя);
- формирование отчетности для диспетчеров и руководства (эффективность курьеров, количество доставок, среднее время).

Система должна стать цифровым ядром логистического отдела компании.

### 2. Описание бизнес-процессов, подлежащих автоматизации

- **Управление клиентской базой и заказами:**
  - Ведение справочников: клиенты (отправители/получатели), типы грузов, тарифы, зоны доставки.
  - Создание заказа: указание точек отправления и назначения, габаритов груза, особых отметок (хрупкий, срочный).
  - Автоматический расчет стоимости доставки (в зависимости от расстояния, веса и срочности).
  - Присвоение уникального трек-номера заказу.

- **Управление курьерами и маршрутами:**
  - Ведение справочника курьеров (транспорт, зона покрытия, график работы).
  - Назначение курьера на заказ (автоматический выбор свободного курьера в зоне или ручной диспетчером).
  - Формирование маршрутного листа курьера на день (список заказов с оптимальной последовательностью).
  - Отслеживание статусов заказа: *Новый → Назначен курьеру → Передан курьеру → Доставляется → Доставлен/Вручен → Завершен*.

- **Учет доставок и подтверждение (QR + геолокация):**
  - Генерация уникального QR-кода для каждого заказа (или сессии доставки) на этапе передачи курьеру.
  - Отображение QR-кода в мобильном приложении курьера для сканирования получателем.
  - Сканирование QR-кода получателем (или курьером при вручении) через мобильное устройство.
  - Автоматическая фиксация времени вручения, геолокации (сравнение с адресом доставки), идентификация получателя.
  - Защита от мошенничества: отметка о получении доступна только в радиусе 100 метров от точки назначения (имитация в настройках), ограничение по времени (не ранее прибытия курьера).
  - Возможность ручного подтверждения диспетчером в случае технических сбоев.

- **Фотофиксация и документооборот:**
  - Возможность загрузки фото груза до отправки и после доставки (подтверждение целостности).
  - Хранение электронных накладных и актов приема-передачи.

- **Личные кабинеты:**

  - **Клиент (Отправитель/Получатель):**
    - Создание заказа (калькулятор стоимости).
    - Отслеживание статуса заказа в реальном времени (трек-номер).
    - История своих доставок.
    - Подтверждение получения (сканирование QR).
    - Чат с диспетчером (опционально).

  - **Курьер:**
    - Просмотр своего расписания/маршрутного листа на день.
    - Получение уведомления о новом заказе.
    - Сканирование QR-кода для старта доставки (взять в работу).
    - Смена статусов заказа («В пути», «Прибыл на место»).
    - Отображение QR-кода получателю для завершения доставки.
    - Загрузка фото подтверждения.

  - **Диспетчер/Логист:**
    - Панель управления всеми активными заказами (доска задач).
    - Назначение курьеров на заказы (drag-and-drop или ручной выбор).
    - Корректировка маршрутов.
    - Контроль времени выполнения заказов (опоздания).
    - Формирование отчетов.

  - **Администратор:**
    - Управление пользователями (курьеры, клиенты, диспетчеры).
    - Ведение справочников (тарифы, типы грузов, зоны доставки).
    - Настройка параметров системы (радиус геозоны для подтверждения).

- **Отчетность и аналитика:**
  - Количество доставок по курьерам за день/неделю.
  - Среднее время доставки (от назначения до вручения).
  - Рейтинг курьеров (по скорости, отзывам).
  - Статистика использования QR-кодов.
  - Финансовая сводка (выручка по тарифам).

### 4. Ключевые требования к базе данных (основной фокус проекта)

Необходимо спроектировать нормализованную реляционную базу данных в MS SQL Server (3NF), отражающую структуру логистического процесса с QR-идентификацией.

**Обязательные сущности:**

- **Пользователи** (общая таблица): UserID, ФИО, Email, Телефон, Хэш пароля, Роль (Client, Courier, Dispatcher, Admin), Активен, Дата регистрации.
- **Клиенты** (расширение Users): Адрес (юридический), Тип (юрлицо/физлицо), ИНН (опционально).
- **Курьеры** (расширение Users): Транспорт (авто/пеший), Госномер, Зона покрытия (ID зоны), График работы, Рейтинг.
- **Типы_грузов**: ID, Название (документы, одежда, техника), Макс_вес, Хрупкий (bit), Описание.
- **Зоны_доставки**: ID, Название, Коэффициент_цены.
- **Тарифы**: ID, Тип_груза, Зона, Цена_за_кг, Цена_за_км, Наценка_срочность.
- **Заказы** (ключевая сущность):
  - OrderID (уникальный трек-номер для клиента).
  - ClientID (отправитель), RecipientName, RecipientPhone, RecipientAddress.
  - Тип_груза, Вес (кг), Объем (опционально).
  - Адрес_подачи, Адрес_доставки (можно геокоординаты).
  - Срочность (bool), Дата_создания, Желаемая_дата_доставки.
  - Статус (enum: Новый, Назначен, Взят_курьером, В_пути, У_получателя, Доставлен, Отменен).
  - Назначенный_курьер (FK), Диспетчер_создавший (FK).
  - Итоговая_стоимость.
- **Сессии_доставки** (аналог QR_сессий):
  - SessionID, OrderID, Дата_генерации, Токен (уникальный для QR), Время_начала_активности (когда курьер взял заказ), Время_окончания_активности (дедлайн).
  - Статус (Активна/Использована/Просрочена).
- **Логистика_маршрутов** (для маршрутных листов):
  - RouteID, CourierID, Дата, Последовательность_заказов (JSON или связующая таблица).
- **История_статусов_заказа** (аудит):
  - ID, OrderID, Статус, Дата_изменения, Геолокация (широта/долгота), Комментарий.
- **Подтверждение_доставки** (фиксация факта вручения):
  - ID, SessionID, OrderID, Время_сканирования, Геолокация_получателя (при скане), IP_адрес, Статус (QR/Ручная), Фото_подтверждения (путь).
  - UNIQUE ограничение: один заказ — одно успешное подтверждение.
- **Уведомления**: ID, UserID, Текст, Дата, Прочитано (для всех ролей).

**Строгие ограничения целостности:**

- **CHECK**: 
  - Вес > 0.
  - Дата_доставки >= Дата_создания.
  - Цена >= 0.
- **UNIQUE**: Email, Телефон (для курьеров и клиентов), Трек-номер заказа (генерируется как уникальный строковый код), Токен сессии.
- **FOREIGN KEY**: Каскадные действия — **NO ACTION** для критичных данных (запрет удаления курьера с активными заказами).

**Сложные ограничения (триггеры/функции):**

- Запрет изменения статуса заказа на «Доставлен», если геолокация подтверждения не совпадает с адресом доставки (в пределах радиуса).
- Запрет назначения курьера, если он уже имеет 5 активных заказов (имитация загруженности).
- Автоматический расчет стоимости при создании заказа (через функцию).

**Индексы:**

- Поиск заказов по статусу и дате (для дашборда диспетчера).
- Поиск активных сессий по токену (для сканирования QR).
- Поиск по трек-номеру (для клиентского отслеживания).
- Поиск заказов по курьеру и дате.

**Объекты БД (минимум):**

1. **Представление (VIEW)** `vw_ЭффективностьКурьеров` — агрегирует количество доставок за день, среднее время выполнения заказа, процент просрочек по каждому курьеру.
2. **Хранимая процедура** `usp_НазначитьКурьера` — принимает ID заказа, находит ближайшего/свободного курьера в зоне покрытия и назначает, генерирует QR-токен для сессии.
3. **Триггер** на таблицу `История_статусов_заказа` (INSTEAD OF INSERT) — проверяет, что если новый статус = «Доставлен», то существует активная сессия с отметкой о сканировании и геолокация совпадает с целевым адресом. В противном случае — блокирует вставку.
4. **Функция** `fn_РассчитатьСтоимость` — принимает ID типа груза, вес, ID зоны и срочность, возвращает итоговую цену по тарифам.
